8INF874 - Padding Oracle Attack

Machine : Serveur vulnerable (localhost:8080) | API : localhost:5001 | Attaquant : localhost:8081

Le serveur chiffre les donnees en AES-128-CBC

Chaque message est chiffre avec une cle aleatoire generee au demarrage du serveur.
Le serveur renvoie IV || C1 || C2 ... en hex.
Il expose aussi un endpoint /paddingOracle qui indique si le padding est valide.
C'est cet endpoint qui cree la vulnerabilite.

Message a chiffrer :

Texte chiffre (hex) -- transmis sur le reseau :



  
Blocs (16 octets chacun) :

  

    
      
BlocHexDescription

    
    
  


  

  

    Ce ciphertext transite sur le reseau.

    L'attaquant peut le capturer et l'utiliser pour retrouver le clair sans la cle.
